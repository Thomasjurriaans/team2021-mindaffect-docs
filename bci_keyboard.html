bci_keyboard.py¶

This module sets up a keyboard provided any GUI.

It contains a single class Keyboard.

A Keyboard has the following functionality:

  • Clickable keys

  • Word prediction

  • A single keypad supporting ENTER, BACKSPACE, SPACE BAR, A-Z

  • A text field keeping track of the text written so far

A Keyboard has the following layout:

  • A text field keeping track of the text written so far

  • Prediction(key)s which display word predictions for the word being produced.

  • Keypads which hold all those keys also present on a physical keyboard.

class bci_keyboard.Keyboard(facade, style_sheet, window_configs, menu_config, tts_config)¶

Bases: object

Sets up the keyboard functionality.

Parameters

  • facade (framework_facade.FrameworkFacade) – Contains the GUI specific functionality.

  • style_sheet (dict) –

    Style instructions of the form: {

    ’keyboard_size’: (int) * Size of the entire keyboard relative to window size, ‘keypad_size’: (int) * Size of just the keypad relative to window size, ‘text_color’: ((int,int,int)) * Color of the key label as an RGB value, ‘button_color’: ((int,int,int)) * Color of the button as an RGB value, ‘line_color’: ((int,int,int)) * Color of the button’s surrounding line as an RGB value, ‘n_predictions’: (int) * Number of prediction suggestions displayed on top of the keypad, ‘switch_keys”: (list(str)) * Labels of the keys that switch to different windows, ‘punctuation’: (str) * Characters that are considered punctuation.

    }

  • window_configs (dict) – The layouts read from a JSON file used for the creation of windows.

  • tts_config (dict) – Text-to-speech settings are defined in here.
